# Рабочие шаблоны и чек-листы

Эти шаблоны предназначены для того, чтобы вы могли сразу же применить полученные знания на практике. Используйте их как основу для создания собственных документов.

## Шаблон 1: Spec-документ для AI-агента

Скопируйте этот шаблон в свой рабочий документ и заполните его для вашего процесса.

```markdown
# Спецификация (Spec) для AI-агента: [Название процесса]

**Версия:** 1.0
**Дата:** 2024-10-27
**Владелец документа:** [Ваше Имя/Отдел]

## 1. Общее описание и цели

**1.1. Автоматизируемый процесс:**
[Краткое описание процесса своими словами. Какую проблему решаем?]

**1.2. Цели автоматизации (KPIs):**
[Что мы хотим улучшить? Например: Сократить время выполнения с X до Y; Снизить количество ошибок на Z%]

**1.3. Границы процесса:**
- **Триггер (начало):** [Какое событие запускает процесс?]
- **Результат (завершение):** [Что является успешным результатом работы агента?]

## 2. Алгоритм работы

**2.1. Участники и системы:**
- **Пользователи:** [Роли пользователей, взаимодействующих с процессом]
- **Системы:** [Список ПО, баз данных, API, с которыми взаимодействует агент]

**2.2. Пошаговый алгоритм:**
[Подробное описание шагов в формате "Если... то...". Можно в виде списка или таблицы.]

**2.3. Блок-схема процесса:**
[Ссылка на диаграмму или вставленное изображение]

## 3. Требования

**3.1. Требования к данным:**
| Источник | Формат данных | Ожидаемое качество | Действия по очистке |
|---|---|---|---|
| [Напр., CRM] | [Напр., JSON API] | [Напр., 95% полноты] | [Напр., Игнорировать записи без email] |
| | | | |

**3.2. Требования к безопасности:**
- **Аутентификация:** [Как агент получает доступ? API-ключ, логин/пароль и т.д.]
- **Авторизация (Права):** [Какие права нужны агенту в каждой системе? Только чтение? Запись?]
- **Работа с чувствительными данными:** [Нужно ли маскировать/анонимизировать данные? Если да, то какие?]

**3.3. Требования к мониторингу:**
- **Ключевые метрики:** [Перечислить 2-3 главные метрики для дашборда]
- **Система оповещений:**
  - **Событие:** [Напр., Ошибка обработки счета]
  - **Канал:** [Напр., Slack, Email]
  - **Получатель:** [Напр., Команда поддержки, AI Champion]

## 4. План действий при сбоях (Error Handling)

| Тип ошибки | Пример | Стратегия обработки | Получатель оповещения |
|---|---|---|---|
| Техническая | [Напр., API недоступен] | [Напр., Повторить 3 раза с интервалом 5 мин] | [Напр., Тех. команда] |
| Бизнес-логика | [Напр., Не найден контрагент] | [Напр., Эскалация на пользователя] | [Напр., AI Champion] |
| Неизвестная | [Напр., Null Pointer Exception] | [Напр., Аварийная остановка] | [Напр., Тех. команда] |

## 5. Жизненный цикл

**5.1. Среды:**
- **Тестовая среда:** [Адрес, доступы]
- **Рабочая среда (Production):** [Адрес, доступы]

**5.2. Процесс развертывания (Deployment):**
[Как изменения попадают в рабочую среду? Ручное копирование? CI/CD?]

```

## Шаблон 2: Профиль AI Champion

Используйте эту структуру для описания идеального кандидата на роль внутреннего евангелиста AI в вашем отделе.

### Профиль кандидата на роль AI Champion для отдела [Название отдела]

| Критерий | Описание требований |
|---|---|
| **Знание предметной области** | - Должен досконально понимать процесс X, Y, Z.<br>- Знает основные "боли" и неэффективности в работе отдела.<br>- Понимает, как устроено взаимодействие с другими отделами. |
| **Технические навыки** | - Уверенный пользователь ПК и офисных программ.<br>- Проявляет интерес к новым технологиям (не боится пробовать новые программы).<br>- (Желательно) Имеет базовое представление о том, что такое API, JSON. |
| **Личные качества (Soft Skills)** | - **Коммуникабельность:** Умеет объяснять сложные вещи простым языком.<br>- **Эмпатия:** Готов помогать коллегам, терпелив к вопросам.<br>- **Проактивность:** Не ждет указаний, сам предлагает улучшения.<br>- **Авторитет:** Является неформальным лидером, к его мнению прислушиваются. |
| **Мотивация** | - Хочет развиваться в области технологий и автоматизации.<br>- Видит ценность AI для компании и своего отдела.<br>- Готов тратить часть своего времени на обучение коллег и поддержку. |

## Шаблон 4: Матрица оценки эффективности

Используйте эту матрицу для регулярного контроля за эффективностью внедренного AI-агента.

### Матрица эффективности для процесса [Название процесса]

**Период оценки:** [Напр., Октябрь 2024]

| Категория | Метрика | Целевое значение | Фактическое значение | Отклонение | Выводы и действия |
|---|---|---|---|---|---|
| **Бизнес** | Сокращение времени цикла | < 10 мин | 8 мин | -20% | **Успех.** |
| | Снижение уровня ошибок | < 1% | 1.5% | +50% | **Требует анализа.** Проверить, почему ошибки все еще возникают. |
| | Экономия затрат (чел/час в мес.) | > 20 | 25 | +25% | **Успех.** |
| **Технические** | Процент успешных выполнений | > 99% | 98% | -1% | **Требует анализа.** Изучить логи сбоев. |
| | Среднее время выполнения | < 90 сек | 105 сек | +16% | **Требует анализа.** Возможно, есть узкие места в производительности. |
| | Доступность (Uptime) | > 99.9% | 100% | +0.1% | **Успех.** |
