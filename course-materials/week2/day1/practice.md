## Практические задания и упражнения

### Задание 1: Моделирование сущностей бизнеса (Базовый уровень)

- **Цель:** Научиться определять и описывать ключевые сущности своего бизнеса.
- **Инструкция:**
    1. Выберите один отдел или команду в вашей компании.
    2. Определите 4-6 ключевых ролей в этом отделе.
    3. Для каждой роли, используя паттерн из лекции, опишите ее `id`, `name`, `responsibilities` и `reports_to`.
    4. Определите 2-3 ключевых процесса, в которых участвует этот отдел.
    5. Для одного из процессов опишите его `id`, `name`, `owner` и `trigger`.
- **Критерии оценки:**
    - Определено не менее 4 ролей.
    - Роли описаны с использованием как минимум 4 атрибутов из паттерна.
    - Определено не менее 2 процессов.
    - Один процесс описан с использованием как минимум 4 атрибутов из паттерна.
- **Ожидаемый результат:** Markdown-файл с описанием ролей и процессов вашего отдела.
- **Тайминг:** 30 минут.

### Задание 2: Описание метрик (Базовый уровень)

- **Цель:** Научиться формализовать бизнес-метрики для машинного чтения.
- **Инструкция:**
    1. Выберите 2-3 важные метрики, за которые отвечает ваш отдел или вы лично.
    2. Для каждой метрики, используя паттерн из лекции, подробно опишите все атрибуты: `id`, `name`, `description`, `formula`, `data_source`, `owner`.
    3. Подумайте, с какими бизнес-целями (`related_goals`) связана каждая метрика.
- **Критерии оценки:**
    - Описано не менее 2 метрик.
    - Для каждой метрики заполнены все 6 атрибутов.
    - Указаны связанные бизнес-цели.
- **Ожидаемый результат:** Markdown-файл с формализованным описанием ключевых метрик.
- **Тайминг:** 25 минут.

### Задание 3: Декомпозиция процесса (Продвинутый уровень)

- **Цель:** Научиться детально описывать бизнес-процесс, готовя его к автоматизации.
- **Инструкция:**
    1. Возьмите один из процессов, определенных в Задании 1.
    2. Декомпозируйте его на 5-7 ключевых шагов (`steps`).
    3. Для каждого шага подробно опишите: `name`, `responsible_role`, `input`, `output`, `tools`.
    4. Проверьте логическую связность шагов: `output` одного шага должен служить `input`-ом для следующего.
- **Критерии оценки:**
    - Процесс разбит на 5-7 логичных шагов.
    - Для каждого шага заполнены все 5 атрибутов.
    - Прослеживается четкая связь между входами и выходами шагов.
- **Ожидаемый результат:** Полное машиночитаемое описание одного бизнес-процесса в формате JSON или YAML.
- **Тайминг:** 40 минут.

### Задание 4: Построение фрагмента контекстного графа (Продвинутый уровень)

- **Цель:** Научиться визуализировать и строить связи между различными бизнес-сущностями.
- **Инструкция:**
    1. Возьмите результаты Заданий 1, 2 и 3.
    2. Используя любой инструмент для построения диаграмм (Miro, Lucidchart, draw.io или даже просто текст), изобразите граф, связывающий ваши сущности.
    3. На графе должны быть узлы (Роли, Процессы, Метрики) и ребра (отношения между ними, например, "отвечает за", "выполняет шаг", "влияет на").
    4. Пример ребер:
        - `[Роль: Product Manager]` -> `(отвечает за)` -> `[Метрика: User Adoption]`
        - `[Роль: Developer]` -> `(выполняет шаг)` -> `[Процесс: Feature Release, Шаг: Code Review]`
        - `[Процесс: Incident Response]` -> `(влияет на)` -> `[Метрика: API Uptime]`
- **Критерии оценки:**
    - На графе представлено не менее 8 узлов (сущностей).
    - Установлено не менее 7 осмысленных связей (ребер) между узлами.
    - Граф наглядно демонстрирует взаимосвязь между ролями, процессами и метриками.
- **Ожидаемый результат:** Схема или текстовое описание фрагмента контекстного графа вашей компании.
- **Тайминг:** 25 минут.
