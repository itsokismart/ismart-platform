## Рабочие шаблоны и чек-листы: Безопасность, права доступа и правила для AI

Этот файл содержит шаблоны и фреймворки, которые помогут вам применить на практике концепции, изученные в ходе сессии.

---

### Шаблон 1: Таблица для проектирования ролей (RBAC)

Используйте эту таблицу для определения ролей AI-агентов, их обязанностей и прав доступа в соответствии с принципом минимальных привилегий.

**Инструкция:** Для каждого нового AI-агента в вашей системе, скопируйте и заполните эту таблицу.

| Параметр | Описание |
| :--- | :--- |
| **Название роли** | Краткое, понятное имя роли (например, `Sales_Analyst_Agent`, `HR_Onboarding_Bot`). |
| **Бизнес-цель** | Какую основную задачу решает агент с этой ролью? (например, «Анализ воронок продаж», «Автоматизация онбординга новых сотрудников»). |
| **Ключевые задачи** | Перечислите 2-4 основные подзадачи, которые выполняет агент (например, «Выполнение SQL-запросов к базе данных продаж», «Создание учетных записей в Active Directory»). |
| **Доступ к системам** | Перечислите все IT-системы, к которым требуется доступ (например, `CRM (Salesforce)`, `База знаний (Confluence)`, `HR-система (Workday)`). |
| **Доступ к данным** | Опишите, к каким конкретно данным или типам данных нужен доступ (например, «Таблица `orders` в базе данных `sales_db`», «Личные дела новых сотрудников»). |
| **Тип доступа** | Укажите необходимый уровень разрешений для каждого ресурса (например, `Read-Only`, `Read/Write`, `Create/Delete`). |
| **Ответственный** | Укажите команду или сотрудника, ответственного за аудит и управление этой ролью. |

### Шаблон 2: Матрица контроля «Автономность vs. Чувствительность»

Используйте эту матрицу для определения необходимого уровня контроля над AI-агентом на основе его автономности и чувствительности данных, с которыми он работает.

**Инструкция:** Найдите ячейку на пересечении строки, соответствующей уровню автономности вашего агента, и столбца, соответствующего уровню чувствительности данных. Эта ячейка определит рекомендуемый уровень контроля.

| | **Низкая чувствительность** (Публичные, открытые данные) | **Средняя чувствительность** (Внутренние, непубличные данные) | **Высокая чувствительность** (Персональные, финансовые, коммерческая тайна) |
| :--- | :--- | :--- | :--- |
| **Низкая автономия** (Ассистент, выполняет команды) | **Минимальный контроль** (Периодический аудит логов) | **Базовый контроль** (Аудит в реальном времени, выборочная проверка результатов) | **Строгий контроль** (Обязательная предварительная проверка (pre-validation) всех критических действий человеком) |
| **Средняя автономия** (Агент с набором правил) | **Базовый контроль** (Аудит в реальном времени, мониторинг аномалий) | **Строгий контроль** (Предварительная проверка критических действий, регулярный пост-исполнительный аудит) | **Максимальный контроль** (Обязательный Human-in-the-Loop для неоднозначных случаев, полная пре-валидация) |
| **Высокая автономия** (Самообучающийся агент) | **Строгий контроль** (Постоянный мониторинг аномалий, детальный пост-исполнительный аудит) | **Максимальный контроль** (Обязательный HITL, детальный аудит принимаемых решений, «песочница» для тестирования) | **ЗАПРЕЩЕНО / СПЕЦИАЛЬНЫЕ УСЛОВИЯ** (Использование только в изолированных средах (air-gapped), работа с полностью синтетическими данными) |

### Чек-лист 3: План реагирования на инциденты безопасности AI

| **1. Изоляция** | Отозвать права доступа у скомпрометированного агента. | | | |
