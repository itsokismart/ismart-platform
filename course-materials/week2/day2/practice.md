## Практические задания и упражнения: Безопасность, права доступа и правила для AI

### Задание 1 (Базовый уровень): Проектирование RBAC для AI-агентов

**Цель:** Научиться определять и описывать роли для AI-агентов на основе их бизнес-функций.

**Сценарий:** Вы работаете в компании «InnovateRetail», которая хочет внедрить трех AI-агентов:
1.  **Агент поддержки:** отвечает на вопросы клиентов в чате на сайте.
2.  **Агент-аналитик:** помогает отделу маркетинга анализировать данные о продажах.
3.  **Агент по закупкам:** автоматизирует процесс заказа канцтоваров для офиса.

**Пошаговая инструкция:**

1.  Для каждого из трех агентов определите его основную бизнес-цель.
2.  Опишите ключевые задачи, которые должен выполнять каждый агент для достижения своей цели.
3.  Определите, к каким системам и данным (например, CRM, база знаний, складская система, финансовый софт) каждому агенту необходим доступ.
4.  Заполните таблицу RBAC, указав для каждой роли (агента) разрешенные действия и необходимые доступы. Используйте шаблон из файла `templates.md`.

**Ожидаемый результат:** Заполненная таблица с описанием трех ролей для AI-агентов, их прав и доступов.

**Критерии оценки:**
*   Четко определены цели и задачи для каждой роли.
*   Права доступа соответствуют принципу минимальных привилегий (агенту не дано лишних прав).
*   Логично определены необходимые для работы системы и данные.

**Тайминг:** 25 минут.

### Задание 2 (Базовый уровень): Разработка политики анонимизации данных

**Цель:** Научиться определять чувствительные данные и применять к ним адекватные методы анонимизации.

**Сценарий:** Ваш AI-агент поддержки (из Задания 1) использует внешний API на базе LLM для генерации ответов клиентам. В чат с клиентом может попадать следующая информация: ФИО клиента, номер заказа, email, текст обращения, история покупок, последние 4 цифры банковской карты.

**Пошаговая инструкция:**

1.  Проанализируйте перечисленные типы данных и классифицируйте их по уровню чувствительности (высокий, средний, низкий).
2.  Определите, какие из этих данных являются персонально идентифицируемой информацией (PII).
3.  Для каждого типа данных, который будет отправляться во внешний API, выберите подходящий метод анонимизации (маскирование, псевдонимизация, обобщение).
4.  Опишите в виде формальной политики, как должен обрабатываться каждый тип данных перед отправкой. Например: «Перед отправкой текста обращения во внешний API, все упоминания ФИО должны быть заменены на `[CUSTOMER_NAME]`, а email — на `[CUSTOMER_EMAIL]`».

**Ожидаемый результат:** Документ с политикой анонимизации, описывающий правила обработки каждого типа данных перед отправкой во внешний сервис.

**Критерии оценки:**
*   Корректно определены PII и уровень чувствительности данных.
*   Выбранные методы анонимизации адекватны и минимизируют риски утечки.
*   Политика написана четко и не допускает двойных толкований.

**Тайминг:** 20 минут.

### Задание 3 (Продвинутый уровень): Проектирование протокола эскалации и Human-in-the-Loop

**Цель:** Разработать детальный протокол взаимодействия AI-агента и человека для сложных и неоднозначных ситуаций.

**Сценарий:** AI-агент по закупкам (из Задания 1) обнаружил, что цена на привычную марку бумаги для принтера выросла на 40%. Стандартный лимит на автоматическую закупку превышен. Кроме того, система поставщика вернула ошибку «Товар временно недоступен» для заказанных ручек.

**Пошаговая инструкция:**

1.  Определите триггеры для эскалации в данном сценарии (превышение лимита, ошибка API).
2.  Опишите пошаговый протокол эскалации: к кому должен обратиться агент (например, к офис-менеджеру), в каком формате передать информацию (сообщение в Slack, email, тикет в Jira), какую информацию должен содержать запрос (суть проблемы, варианты решения).
3.  Разработайте сценарий Human-in-the-Loop. Опишите, какие действия должен предпринять человек (офис-менеджер) после получения эскалации.
4.  Продумайте, как решение, принятое человеком, может быть использовано для дообучения агента в будущем (например, если человек одобрит закупку по новой цене, агент может повысить свой лимит для этого товара на 5%).

**Ожидаемый результат:** Описание протокола эскалации и сценария HITL для AI-агента по закупкам.

**Критерии оценки:**
*   Четко определены триггеры и получатели эскалации.
*   Протокол эскалации детально описывает процесс взаимодействия.
*   Сценарий HITL реалистичен и включает механизм обратной связи для дообучения агента.

**Тайминг:** 30 минут.

### Задание 4 (Продвинутый уровень): Разработка матрицы контроля и плана реагирования на инциденты

**Цель:** Научиться оценивать риски и создавать формализованные планы для управления безопасностью AI.

**Сценарий:** Ваша компания разрабатывает нового **AI-агента для HR-департамента**. Его задача — анализировать резюме кандидатов и проводить первичный скрининг. Агент будет иметь доступ к базе резюме, содержащей ФИО, контактные данные, опыт работы и зарплатные ожидания.

**Пошаговая инструкция:**

1.  **Матрица контроля:**
    *   Определите уровень чувствительности данных, с которыми работает агент (по шкале: низкая, средняя, высокая).
    *   Определите предполагаемый уровень автономии агента (по шкале: низкая, средняя, высокая).
    *   Используя матрицу из лекции (или шаблон из `templates.md`), определите требуемый уровень контроля для данного агента.
    *   Кратко обоснуйте свой выбор.

2.  **План реагирования на инциденты:**
    *   Разработайте краткий план реагирования на инцидент по сценарию: «Обнаружено, что из-за уязвимости агент получил доступ к зарплатной ведомости всех сотрудников компании».
    *   Опишите действия для каждого из 6 этапов: Изоляция, Анализ, Уведомление, Устранение, Восстановление, Извлечение уроков.

**Ожидаемый результат:**
1.  Заполненная матрица контроля с обоснованием.
2.  Краткий, но четкий план реагирования на инцидент безопасности.

**Критерии оценки:**
*   Адекватно оценены чувствительность данных и автономия агента.
*   Выбранный уровень контроля логично вытекает из матрицы.
*   План реагирования на инцидент включает все 6 ключевых этапов и содержит конкретные, релевантные действия для каждого этапа.

**Тайминг:** 35 минут.
