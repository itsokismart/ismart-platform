# Практические задания и упражнения

Этот блок заданий поможет вам на практике освоить инструменты и концепции, рассмотренные в лекции. Выполняя их, вы заложите фундамент для создания своего персонального AI-ассистента.

---

### Задание 1: Настройка рабочей среды (Базовый уровень)

**Цель:** Установить и настроить основные инструменты — Cursor и n8n.

**Тайминг:** 45 минут

**Пошаговая инструкция:**

1.  **Установка Cursor:**
    *   Перейдите на официальный сайт [Cursor](https://cursor.com/).
    *   Скачайте и установите версию для вашей операционной системы (Windows, macOS, Linux).
    *   Запустите Cursor. Ознакомьтесь с интерфейсом. Обратите внимание, что он очень похож на VS Code.
    *   Попробуйте открыть встроенный чат (обычно `Ctrl+K` или `Cmd+K`) и задать AI-ассистенту простой вопрос, например: «What is your name?».

2.  **Регистрация и настройка n8n Cloud:**
    *   Перейдите на сайт [n8n.io](https://n8n.io/) и зарегистрируйте бесплатный аккаунт в облачной версии.
    *   После регистрации вы попадете в панель управления. Создайте свой первый workflow.
    *   Ознакомьтесь с интерфейсом: слева — панель с нодами (узлами), в центре — холст для построения workflow.

**Ожидаемые результаты:**

*   На вашем компьютере установлен и запущен Cursor IDE.
*   Вы успешно зарегистрировали аккаунт в n8n Cloud и можете создавать новые workflow.

**Критерии оценки:**

*   **Выполнено:** Оба инструмента установлены/настроены и готовы к работе.
*   **Не выполнено:** Хотя бы один из инструментов не установлен или не настроен.

### Задание 2: Первые шаги в Cursor (Базовый уровень)

**Цель:** Научиться использовать базовые AI-функции Cursor для генерации и редактирования кода.

**Тайминг:** 30 минут

**Пошаговая инструкция:**

1.  Создайте новый файл в Cursor с названием `hello.py`.
2.  Откройте чат с AI (`Ctrl+L` или `Cmd+L`) и попросите его: `«Напиши простую функцию на Python, которая принимает имя в качестве аргумента и выводит приветствие 'Hello, [имя]!'»`.
3.  Вставьте сгенерированный код в файл `hello.py`.
4.  Теперь выделите сгенерированную функцию и снова откройте чат. Попросите AI: `«Добавь в эту функцию проверку, чтобы если имя не передано, она выводила 'Hello, World!'»`.
5.  Изучите изменения, которые внес AI.
6.  Попробуйте «сломать» код (например, удалите двоеточие) и используйте функцию `Fix with AI` для автоматического исправления.

**Ожидаемые результаты:**

*   Файл `hello.py` с работающей функцией приветствия, которая корректно обрабатывает отсутствие имени.
*   Вы понимаете, как использовать чат для генерации и рефакторинга кода.

**Критерии оценки:**

*   **Выполнено:** Функция в `hello.py` работает согласно требованиям. Вы можете продемонстрировать использование чата для генерации и редактирования.
*   **Не выполнено:** Функция не работает или вы не смогли использовать AI-функции для ее создания/изменения.

### Задание 3: Создание первого Workflow в n8n (Продвинутый уровень)

**Цель:** Создать простую, но полезную автоматизацию, связывающую два веб-сервиса.

**Тайминг:** 60 минут

**Пошаговая инструкция:**

1.  **Идея Workflow:** Мы создадим workflow, который каждый день в 9 утра будет проверять погоду в вашем городе и присылать вам уведомление в Telegram (если у вас нет Telegram, можно использовать Email или другой удобный мессенджер, поддерживаемый n8n).

2.  **Настройка триггера:**
    *   В новом workflow добавьте ноду `Schedule`.
    *   Настройте ее на ежедневное срабатывание в 9:00.

3.  **Получение данных о погоде:**
    *   Добавьте ноду `HTTP Request`.
    *   Вам понадобится API для получения погоды. Можно использовать бесплатный сервис, например, Open-Meteo. Изучите их документацию, чтобы составить правильный URL для запроса (он должен включать широту и долготу вашего города).
    *   Настройте ноду `HTTP Request` на отправку GET-запроса по этому URL.

4.  **Отправка уведомления:**
    *   Добавьте ноду для вашего мессенджера (например, `Telegram`).
    *   Вам нужно будет создать бота в Telegram и получить его токен. Следуйте инструкциям в документации n8n для ноды Telegram.
    *   В тексте сообщения используйте данные из предыдущей ноды (HTTP Request). С помощью Expression Editor в n8n вы можете составить сообщение вида: `«Доброе утро! Сегодня в вашем городе температура: [температура] градусов.»`.

5.  **Тестирование и активация:**
    *   Протестируйте каждую ноду по отдельности, а затем весь workflow целиком.
    *   После успешного тестирования активируйте workflow.

**Ожидаемые результаты:**

*   Активный workflow в n8n, который ежедневно присылает вам актуальный прогноз погоды.

**Критерии оценки:**

*   **Выполнено:** Workflow работает корректно и присылает осмысленные уведомления.
*   **Частично выполнено:** Workflow создан, но работает с ошибками или не активирован.
*   **Не выполнено:** Задание не начато или workflow не создан.

### Задание 4: Создание персональной базы знаний с RAG (Продвинутый уровень)

**Цель:** Понять на практике, как работает RAG, создав простую систему вопросов и ответов на основе своих документов.

**Тайминг:** 90 минут

**Пошаговая инструкция:**

1.  **Подготовка данных:**
    *   Соберите 3-5 документов в формате `.txt` или `.md`, на основе которых вы хотите построить свою базу знаний. Это могут быть ваши рабочие инструкции, статьи по интересной вам теме, личные заметки.
    *   Сложите все файлы в одну папку в Cursor.

2.  **Использование AI-чата с контекстом папки:**
    *   В Cursor есть функция, позволяющая «скормить» AI всю папку как контекст. Обычно это делается через `@-команды` в чате (например, `@workspace` или `@folder`).
    *   Выберите вашу папку с документами в качестве контекста для чата.

3.  **Тестирование RAG:**
    *   Теперь задайте AI-ассистенту вопрос, ответ на который точно содержится в ваших документах, но вряд ли известен модели из ее общих знаний. Например: `«Каковы ключевые шаги в процессе [название вашего процесса из документа]?»`.
    *   Проанализируйте ответ. Обратите внимание, ссылается ли модель на ваши документы.
    *   Задайте еще несколько каверзных вопросов, чтобы проверить, насколько хорошо система находит информацию.
    *   Попросите модель сделать саммари одного из ваших документов, не указывая его название, а лишь описав его содержание.

**Ожидаемые результаты:**

*   Вы умеете использовать функцию предоставления папки в качестве контекста в Cursor.
*   Вы получаете точные, основанные на ваших документах ответы от AI-ассистента.

**Критерии оценки:**

*   **Выполнено:** Вы успешно задали 3+ вопроса к своей базе знаний и получили релевантные ответы, основанные на предоставленных документах.
*   **Не выполнено:** Вы не смогли получить от AI ответы, основанные на ваших документах.

### Задание 5: Сборка Персонального Ассистента (Комплексное задание)

**Цель:** Интегрировать полученные знания и создать прототип персонального ассистента с кастомным навыком.

**Тайминг:** 120 минут

**Пошаговая инструкция:**

1.  **Определение задачи для навыка:**
    *   Выберите одну рутинную задачу, которую вы часто выполняете. Например: «создание отчета по итогам недели», «написание follow-up письма после встречи», «структурирование заметок».

2.  **Проектирование промпта для навыка:**
    *   В отдельном файле `my_skill.md` в Cursor подробно опишите промпт для вашего навыка. Используйте шаблоны из лекции.
    *   Ваш промпт должен быть четким, структурированным и включать переменные (плейсхолдеры) для входных данных (например, `{{тема_встречи}}`, `{{ключевые_решения}}`).

3.  **Интеграция с n8n (опционально, для продвинутых):**
    *   Создайте в n8n новый workflow, который будет вызываться по вебхуку (Webhook).
    *   Этот workflow будет принимать данные (например, в формате JSON), необходимые для вашего навыка.
    *   Добавьте ноду, которая будет отправлять эти данные в языковую модель (например, через API OpenAI), используя ваш промпт из `my_skill.md`.
    *   Результат работы модели workflow может отправить вам в почту или мессенджер.

4.  **Тестирование навыка:**
    *   **Базовый вариант:** Скопируйте ваш промпт из `my_skill.md` в чат Cursor, вручную подставьте в него реальные данные и проверьте результат.
    *   **Продвинутый вариант:** Отправьте тестовый запрос на ваш вебхук в n8n и убедитесь, что вся цепочка работает и вы получаете ожидаемый результат в нужном месте.

**Ожидаемые результаты:**

*   Готовый, хорошо описанный промпт для кастомного навыка в файле `my_skill.md`.
*   (Продвинутый) Работающий workflow в n8n, который автоматизирует выполнение вашего навыка.

**Критерии оценки:**

*   **Отлично:** Создан и протестирован промпт, реализована и работает интеграция с n8n.
*   **Хорошо:** Создан и протестирован качественный промпт для навыка (без интеграции с n8n).
*   **Удовлетворительно:** Создан черновик промпта, но он требует доработки.
*   **Не выполнено:** Задание не начато.
