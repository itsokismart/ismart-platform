# Рабочие шаблоны и чек-листы

Этот раздел содержит полезные шаблоны, фреймворки и чек-листы, которые помогут вам в проектировании и создании ваших персональных AI-ассистентов и автоматизаций.

---

### Шаблон 1: Проектирование Кастомного Навыка (Skill)

Используйте эту таблицу для детального проектирования ваших кастомных навыков перед тем, как писать промпт.

| Компонент | Описание |
| --- | --- |
| **Название навыка** | Краткое, понятное имя (например, `WeeklyReportGenerator`). |
| **Цель навыка** | Что должен делать этот навык? Какую проблему он решает? (например, `Автоматически генерировать еженедельный отчет по продажам на основе данных из CRM`). |
| **Триггер** | Какое событие запускает этот навык? (например, `Каждую пятницу в 17:00`, `По HTTP-запросу от n8n`, `При добавлении нового файла в папку`). |
| **Входные данные (Inputs)** | Какая информация нужна навыку для работы? (плейсхолдеры в промпте). |
| *Пример 1:* | `{{период_отчета}}` (например, `10.10.2025 - 17.10.2025`). |
| *Пример 2:* | `{{данные_о_сделках_json}}` (структурированные данные из CRM). |
| *Пример 3:* | `{{имя_менеджера}}`. |
| **Выходные данные (Output)** | Что навык должен вернуть в результате? В каком формате? |
| *Пример:* | `Markdown-файл с отчетом, включающий саммари, ключевые метрики и список топ-5 сделок.` |
| **Основной промпт (Core Prompt)** | Здесь пишется сам промпт, включая роль, инструкции, контекст и переменные. |
| **Зависимости / Интеграции** | Какие внешние инструменты или API нужны этому навыку? (например, `API CRM-системы`, `n8n workflow`, `Google Sheets API`). |

### Шаблон 2: Фреймворк для RAG-системы

Этот фреймворк поможет вам структурировать процесс создания и оценки вашей Retrieval-Augmented Generation системы.

| Этап | Ключевые вопросы и действия |
| --- | --- |
| **1. Определение Источников (Source Definition)** | - Какие документы и данные будут служить базой знаний? (PDF, .txt, .md, URL, Notion, etc.)<br>- Каков объем этих данных?<br>- Как часто они обновляются? |
| **2. Подготовка и Чанкинг (Preparation & Chunking)** | - Нужно ли предварительно очищать данные? (удалять "мусор", форматирование).<br>- Какой размер чанка (фрагмента) будет оптимальным? (например, 256, 512, 1024 токена).<br>- Будет ли использоваться перекрытие (overlap) между чанками для сохранения контекста? |
| **3. Векторизация (Vectorization)** | - Какая модель эмбеддингов будет использоваться? (зависит от языка, предметной области).<br>- Где будут храниться векторы? (локально, в облачной векторной БД, например, Pinecone, Weaviate). |
| **4. Процесс Поиска (Retrieval Process)** | - Какой метод поиска будет использоваться? (поиск по сходству, MMR - Maximal Marginal Relevance).<br>- Сколько релевантных чанков (top-k) будет извлекаться для одного запроса? |
| **5. Генерация Ответа (Generation)** | - Какая LLM будет использоваться для генерации финального ответа?<br>- Как будет выглядеть промпт, объединяющий исходный вопрос пользователя и найденные чанки? |
| **6. Оценка Качества (Evaluation)** | - Как будет измеряться релевантность найденных чанков?<br>- Как будет измеряться точность и полнота финального ответа?<br>- Подготовить набор тестовых вопросов и эталонных ответов для оценки. |

### Чек-лист 3: Готовность Персонального Ассистента

Используйте этот чек-лист для самопроверки перед завершением работы над вашим персональным ассистентом в рамках подготовительного модуля.

**Среда и Инструменты:**
- [ ] Cursor IDE установлен и настроен.
- [ ] Аккаунт n8n (Cloud или Self-hosted) создан и доступен.
- [ ] Вы умеете создавать и редактировать файлы в Cursor.
- [ ] Вы умеете создавать, настраивать и активировать workflow в n8n.

**Контекст и Память (RAG):**
- [ ] Подготовлен набор документов (минимум 3) для вашей персональной базы знаний.
- [ ] Вы успешно использовали контекст папки/файлов в Cursor для ответов на вопросы.
- [ ] Вы можете получить от AI ответ, основанный исключительно на ваших документах.

**Навыки (Skills):**
- [ ] Спроектирован как минимум один кастомный навык с использованием «Шаблона 1».
- [ ] Написан и протестирован в чате Cursor промпт для этого навыка.
- [ ] (Продвинутый) Создан workflow в n8n, который автоматизирует вызов этого навыка.

**Интеграции:**
- [ ] Вы понимаете, как работает нода `HTTP Request` в n8n.
- [ ] Вы понимаете, как передавать данные между нодами в n8n.
- [ ] (Продвинутый) Ваш workflow в n8n интегрирован как минимум с одним внешним сервисом (погода, мессенджер, email).

**Итоговый Результат:**
- [ ] У вас есть работающий прототип AI-ассистента, который может выполнять как минимум одну полезную для вас задачу с помощью кастомного навыка.
- [ ] Вы понимаете, как дальше развивать вашего ассистента, добавляя новые навыки и интеграции.
