# Дизайн-система «Цифровой атлас знаний» (Knowledge Atlas)

## Обзор подхода

**Knowledge Atlas** — это комплексный дизайн-подход, объединяющий **Editorial Design** (журнальная вёрстка) и **Swiss Typography** (швейцарский стиль типографики). Система предназначена для создания образовательных платформ, портальов знаний, документации и интерактивных курсов, которые ощущаются как **премиум-издание** — полированные, авторитетные и легко навигируемые.

### Ключевые характеристики

- **Журнальная вёрстка**: асимметричные макеты, контрастные размеры типографики, белое пространство как активный элемент дизайна
- **Швейцарская типография**: чёткая иерархия, сетка-основа, минимум орнамента, максимум функциональности
- **Цветовая кодификация**: семантические цвета для категорий, модулей и типов контента
- **Интерактивные элементы**: плавные переходы, микровзаимодействия, визуальная обратная связь
- **Доступность**: контрастные цвета, читаемая типография, логичная навигация

## 2. Типографика

### 2.1 Шрифтовая система

#### Display Font (Заголовки)
- **Шрифт**: `Playfair Display` (Google Fonts)
- **Размеры**: 
  - H1: 48px (desktop) / 36px (mobile) — вес 700
  - H2: 36px (desktop) / 28px (mobile) — вес 700
  - H3: 28px (desktop) / 22px (mobile) — вес 600
- **Характеристики**: Высокий контраст, элегантные засечки, создаёт премиум-ощущение
- **Применение**: Заголовки страниц, названия модулей, ключевые фразы

#### Body Font (Основной текст)
- **Шрифт**: `Inter` (Google Fonts)
- **Размеры**:
  - Body: 16px — вес 400 (line-height: 1.6)
  - Small: 14px — вес 400 (line-height: 1.5)
  - Tiny: 12px — вес 500 (line-height: 1.4)
- **Характеристики**: Геометрический гротеск, отличная читаемость, нейтральный
- **Применение**: Основной текст, описания, инструкции

#### Mono Font (Код и данные)
- **Шрифт**: `Fira Code` (Google Fonts)
- **Размер**: 13px — вес 400 (line-height: 1.7)
- **Применение**: Блоки кода, терминальные команды, переменные

### 2.2 Иерархия типографики

```
H1 (Playfair Display, 48px, 700)
  ↓ Основной заголовок страницы
  
H2 (Playfair Display, 36px, 700)
  ↓ Заголовки разделов
  
H3 (Playfair Display, 28px, 600)
  ↓ Подзаголовки, названия блоков
  
Body (Inter, 16px, 400)
  ↓ Основной текст
  
Small (Inter, 14px, 400)
  ↓ Вторичный текст, подписи
  
Tiny (Inter, 12px, 500)
  ↓ Метаинформация, теги
```

### 2.3 Правила типографики

1. **Контраст размеров**: Разница между заголовком и телом текста должна быть минимум 1.5x (48px vs 16px = 3x)
2. **Межстрочный интервал**: 1.6 для основного текста, 1.4 для кода
3. **Длина строки**: 65-75 символов для оптимальной читаемости
4. **Вес шрифта**: Используйте только 400, 500, 600, 700 — избегайте промежуточных весов
5. **Капитализация**: Только для аббревиатур и специальных случаев; основной текст — sentence case

## 3. Макетная сетка и пространство

### 3.1 Сетка (Grid System)

- **Базовая единица**: 4px
- **Контейнер**: max-width 1280px (desktop), 100% (mobile)
- **Padding контейнера**: 16px (mobile), 24px (tablet), 32px (desktop)
- **Колонки**: 12-колонная сетка (CSS Grid или Tailwind)
- **Гаттер (промежуток между колонками)**: 24px

### 3.2 Система отступов (Spacing Scale)

| Размер | Px | Применение |
|--------|----|----|
| **xs** | 4px | Микро-отступы внутри компонентов |
| **sm** | 8px | Отступы между элементами одной группы |
| **md** | 16px | Стандартный отступ между блоками |
| **lg** | 24px | Отступы между секциями |
| **xl** | 32px | Отступы между крупными блоками |
| **2xl** | 48px | Отступы между основными разделами |
| **3xl** | 64px | Отступы между модулями |

**Правило**: Всегда используйте значения из шкалы, кратные 4px. Не добавляйте произвольные отступы.

### 3.3 Белое пространство как дизайн-элемент

- **Минимум 40% страницы должно быть пусто** (белое пространство)
- **Вокруг заголовков**: 24px сверху, 16px снизу
- **Вокруг блоков кода**: 16px со всех сторон
- **Вокруг таблиц**: 24px сверху и снизу
- **Между секциями**: минимум 48px

## 5. Анимации и микровзаимодействия

### 5.1 Переходы (Transitions)

| Элемент | Длительность | Easing | Эффект |
|---------|--------------|--------|--------|
| **Hover на кнопку** | 200ms | ease-out | scale(1.05) + shadow |
| **Fade-in контента** | 500ms | ease-out | opacity 0→1 |
| **Scroll-to-section** | 600ms | ease-in-out | smooth scroll |
| **Progress bar** | 500ms | ease-out | width animation |
| **Modal открытие** | 300ms | ease-out | scale + fade |

### 5.2 Микровзаимодействия

**Кнопка "Пройдено"**:
- Нажатие: scale(0.95) + ripple эффект
- Завершение: checkmark animation + color change
- Duration: 400ms total

**Прогресс-бар**:
- Обновление: плавная анимация ширины (500ms)
- Цвет: переход от `muted` к цвету модуля

**Карточка модуля**:
- Hover: поднятие (translateY -4px) + shadow increase
- Duration: 200ms

### 5.3 Принципы анимации

1. **Не более 500ms** для основных взаимодействий
2. **ease-out** для появления, **ease-in-out** для перемещения
3. **Не анимируйте** более 2-3 свойств одновременно
4. **Respect prefers-reduced-motion** для доступности

## 9. Заключение

**Knowledge Atlas** — это не просто набор цветов и шрифтов, а **философия дизайна**, которая превращает информацию в **авторитетное, красивое и доступное издание**. Применяя эту систему, вы создаёте интерфейсы, которые:

- **Уважают пользователя**: чёткая иерархия, логичная навигация, минимум отвлечений
- **Вызывают доверие**: премиум-ощущение через типографику и пространство
- **Легко масштабируются**: модульная система позволяет добавлять новые категории без переделки
- **Остаются актуальными**: классический дизайн не выходит из моды

Используйте этот гайд как **основу**, а не как **догму**. Адаптируйте цвета, шрифты и компоненты под вашу аудиторию и контент, но сохраняйте **ключевые принципы**: иерархию, пространство, контраст и функциональность.
