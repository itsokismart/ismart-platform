## Практические задания и упражнения

### Задание 1: Установка и конфигурация Cursor

*   **Уровень сложности:** Базовый
*   **Тайминг:** 20 минут

**Описание:** Cursor — это форк VS Code, специально адаптированный для работы с AI. Он позволяет редактировать код в диалоговом режиме с AI, что значительно ускоряет разработку. На этом задании вы установите и настроите свой первый AI--редактор.

**Пошаговая инструкция:**

1.  Перейдите на официальный сайт [Cursor](https://cursor.sh/).
2.  Скачайте дистрибутив для вашей операционной системы (Windows, macOS, Linux).
3.  Установите приложение, следуя инструкциям инсталлятора.
4.  Запустите Cursor. При первом запуске вам будет предложено настроить AI-провайдера. Вы можете использовать свой OpenAI API ключ или войти с помощью Google-аккаунта для использования базовых моделей.
5.  Создайте новый файл с расширением `.py` (например, `hello.py`).
6.  Напишите в файле `print("Hello, AI-Native World!")`.
7.  Откройте встроенный терминал (View -> Terminal) и выполните команду `python hello.py`.

**Критерии оценки:**

*   Cursor успешно установлен и запускается.
*   Вы смогли создать, сохранить и выполнить Python-скрипт.

**Ожидаемый результат:**

В терминале выведено сообщение `Hello, AI-Native World!`.

### Задание 2: Подключение MCP серверов

*   **Уровень сложности:** Продвинутый
*   **Тайминг:** 45 минут

**Описание:** MCP (Model Context Protocol) — это протокол, который позволяет AI-агентам безопасно взаимодействовать с внешними API и базами данных. В этом задании вы научитесь подключать публичные MCP-серверы к вашему агенту для получения данных из внешних источников.

**Пошаговая инструкция:**

1.  Ознакомьтесь с концепцией MCP. Изучите [документацию](https://github.com/manus-team/mcp) (гипотетическая ссылка), чтобы понять, как MCP-серверы предоставляют контекст для моделей.
2.  Выберите публичный MCP-сервер для эксперимента. Например, сервер, предоставляющий данные о погоде.
3.  В настройках вашего AI-агента (или в коде, если вы создаете его с нуля) необходимо указать эндпоинт (URL) выбранного MCP-сервера.
4.  Сформулируйте запрос к агенту, который требует обращения к данным с MCP-сервера. Например: «Какая сейчас погода в Москве?». Агент должен понять, что для ответа на этот вопрос нужно обратиться к инструменту, связанному с MCP-сервером погоды.
5.  Проанализируйте ответ агента. Он должен содержать актуальные данные, полученные от внешнего API через MCP.

**Критерии оценки:**

*   Агент успешно подключается к MCP-серверу.
*   Агент правильно интерпретирует запрос пользователя и использует соответствующий инструмент.
*   Агент возвращает корректные данные из внешнего источника.

**Ожидаемый результат:**

На запрос о погоде агент возвращает актуальный прогноз, например: «В Москве сейчас +15 градусов, облачно».

### Задание 3: Разработка набора базовых скиллов

*   **Уровень сложности:** Продвинутый
*   **Тайминг:** 60 минут

**Описание:** Скиллы (навыки или инструменты) — это то, что делает вашего AI-агента по-настоящему полезным. Это функции, которые агент может выполнять в ответ на ваши команды. В этом задании вы создадите свой первый кастомный скилл.

**Пошаговая инструкция:**

1.  Определите простую, но полезную задачу, которую вы часто выполняете. Например, поиск определения термина в Википедии.
2.  Создайте новый Python-файл, например, `wikipedia_skill.py`.
3.  Напишите функцию, которая принимает на вход поисковый запрос (термин), обращается к API Википедии, находит статью и возвращает ее краткое содержание.
    ```python
    import wikipedia

    def get_wikipedia_summary(term):
        try:
            # Устанавливаем русский язык
            wikipedia.set_lang("ru")
            summary = wikipedia.summary(term, sentences=2)
            return summary
        except wikipedia.exceptions.PageError:
            return f"Статья о ''{term}'' не найдена."
        except Exception as e:
            return f"Произошла ошибка: {e}"
    ```
4.  Опишите этот скилл в формате, понятном вашему агенту (обычно это JSON- или YAML-описание, где указано имя функции, ее параметры и описание).
5.  Подключите этот скилл к вашему агенту. Теперь вы можете делать запросы вида: «Что такое 'искусственный интеллект' по Википедии?».

**Критерии оценки:**

*   Функция-скилл работает корректно и возвращает ожидаемый результат.
*   Агент правильно распознает команду и вызывает соответствующий скилл.

**Ожидаемый результат:**

Агент в ответ на ваш запрос возвращает краткое определение термина из Википедии.

### Задание 4: Настройка n8n workflow для автоматизации рутинных процессов

*   **Уровень сложности:** Продвинутый
*   **Тайминг:** 60 минут

**Описание:** n8n — это инструмент для автоматизации рабочих процессов с открытым исходным кодом. Он позволяет связывать различные приложения и сервисы без написания кода. В этом задании вы создадите workflow, который будет запускать вашего AI-агента по определенному триггеру.

**Пошаговая инструкция:**

1.  Зарегистрируйтесь в облачной версии n8n или установите его локально.
2.  Создайте новый workflow (рабочий процесс).
3.  В качестве триггера выберите «Webhook». n8n сгенерирует уникальный URL.
4.  Добавьте следующий узел — «HTTP Request». Настройте его так, чтобы он отправлял POST-запрос на API вашего AI-агента. В теле запроса передавайте данные, полученные вебхуком.
5.  Активируйте workflow.
6.  С помощью любого инструмента (например, `curl` или Postman) отправьте тестовый POST-запрос на URL вашего вебхука, передав в теле JSON с командой для агента. Например: `{"command": "Отправь мне на почту сводку новостей за сегодня"}`.
7.  Проверьте, что ваш AI-агент получил команду и выполнил ее.

**Критерии оценки:**

*   Workflow в n8n успешно создан и активирован.
*   Вебхук принимает данные и передает их в узел HTTP Request.
*   AI-агент получает команду от n8n и выполняет ее.

**Ожидаемый результат:**

После отправки запроса на вебхук, ваш AI-агент выполняет заданное действие (например, вы получаете email со сводкой новостей). Это демонстрирует возможность запускать сложные AI-процессы по внешним событиям.
