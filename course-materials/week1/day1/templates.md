## Рабочие шаблоны и чек-листы

### Шаблон 1: Проектирование системного промпта для AI-агента

Используйте эту таблицу для структурированного создания системного промпта для вашего персонального AI-агента. Заполните каждый раздел, чтобы четко определить его роль, возможности и ограничения.

| Компонент промпта | Описание | Пример заполнения |
| --- | --- | --- |
| **Роль (Role)** | Кем является агент? Какова его основная функция? | `Ты — мой личный ассистент по продуктивности.` |
| **Цель (Goal)** | Какова главная, измеримая цель агента? | `Твоя цель — помочь мне сэкономить 1 час рабочего времени каждый день за счет автоматизации рутинных задач.` |
| **Контекст (Context)** | Какая информация важна для понимания задач? | `Я — руководитель отдела маркетинга в B2B SaaS компании. Мои основные задачи — ...` |
| **Инструменты (Tools)** | Какие инструменты (скиллы) есть в распоряжении агента? | `У тебя есть доступ к: 1. Календарю Google (чтение, создание событий). 2. Jira (создание задач, чтение статусов). 3. Wikipedia (поиск информации).` |
| **Правила (Rules)** | Какие ограничения и правила должен соблюдать агент? | `1. Никогда не отправляй письма клиентам без моего прямого подтверждения. 2. Всегда отвечай в профессиональном и сдержанном стиле. 3. Если не уверен в ответе, уточни у меня.` |
| **Персона (Persona)** | Какой стиль общения должен использовать агент? | `Твой стиль общения: краткий, по делу, проактивный. Предлагай решения, а не просто констатируй проблемы.` |
| **Формат вывода (Output Format)** | В каком формате агент должен предоставлять информацию? | `Все отчеты предоставляй в виде Markdown таблиц. Списки задач форматируй как чек-листы.` |

### Чек-лист 2: Самопроверка готовности персонального AI-агента

Используйте этот чек-лист для оценки готовности вашего агента к работе. Пройдитесь по каждому пункту, чтобы убедиться, что все ключевые аспекты учтены.

- [ ] **1. Определение цели:**
    - [ ] Четко ли сформулирована главная цель агента?
    - [ ] Можно ли измерить достижение этой цели?
- [ ] **2. Системный промпт:**
    - [ ] Заполнены ли все разделы шаблона системного промпта?
    - [ ] Является ли промпт однозначным и понятным?
- [ ] **3. Набор инструментов (скиллов):**
    - [ ] Установлены ли все необходимые базовые скиллы (коммуникация, аналитика, управление задачами)?
    - [ ] Разработан ли хотя бы один кастомный скилл под ваши уникальные задачи?
    - [ ] Протестирован ли каждый скилл по отдельности?
- [ ] **4. Интеграции:**
    - [ ] Подключены ли необходимые MCP-серверы для доступа к внешним данным?
    - [ ] Настроена ли интеграция с вашими основными рабочими инструментами (календарь, таск-менеджер, почта)?
- [ ] **5. Тестирование:**
    - [ ] Проведены ли тесты на простых, одношаговых задачах?
    - [ ] Проведены ли тесты на сложных, многошаговых задачах?
    - [ ] Проверена ли реакция агента на некорректные или неоднозначные команды?
- [ ] **6. Безопасность и ограничения:**
    - [ ] Четко ли определены в системном промпте действия, которые агент не может выполнять автономно?
    - [ ] Есть ли механизм подтверждения для критически важных операций (например, отправка писем, удаление данных)?

### Фреймворк 3: Идентификация возможностей для автоматизации с помощью AI-агента

Этот фреймворк поможет вам системно проанализировать вашу работу и найти задачи, которые можно эффективно делегировать AI-агенту.

1.  **Аудит задач (1-2 дня):**
    *   В течение двух рабочих дней фиксируйте абсолютно все задачи, которые вы выполняете. Используйте для этого простой текстовый файл или таблицу.
    *   Для каждой задачи отмечайте, сколько времени она заняла.
2.  **Классификация задач:**
    *   Разделите все зафиксированные задачи на 4 категории:
        *   **Рутинные и повторяющиеся:** Задачи, которые вы делаете регулярно по одному и тому же алгоритму (например, составление еженедельного отчета).
        *   **Информационный поиск:** Задачи, связанные с поиском, сбором и структурированием информации (например, поиск контактов, подготовка к встрече).
        *   **Коммуникационные:** Написание писем, ответов, назначение встреч.
        *   **Творческие и стратегические:** Задачи, требующие креативности, принятия решений, разработки стратегии.
3.  **Оценка потенциала автоматизации:**
    *   Оцените каждую задачу из первых трех категорий по двум параметрам:
        *   **Частота:** Как часто вы выполняете эту задачу (раз в день, раз в неделю, раз в месяц)?
        *   **Трудоемкость:** Сколько времени она занимает?
4.  **Выбор кандидатов на автоматизацию:**
    *   Задачи с высокой частотой и высокой/средней трудоемкостью — это ваши главные кандидаты на автоматизацию с помощью AI-агента. Начните именно с них, чтобы получить максимальный эффект в кратчайшие сроки.

### Матрица 4: Оценка сложности и влияния внедрения AI-агента

Используйте эту матрицу для приоритизации задач по созданию и внедрению скиллов для вашего AI-агента. Это поможет вам выбрать, с чего начать, чтобы получить быстрые победы и доказать ценность технологии.

| | **Низкая сложность внедрения** | **Высокая сложность внедрения** |
| :--- | :--- | :--- |
| **Высокое влияние на бизнес** | **Быстрые победы (Quick Wins)**<br>_Пример: Автоматическое составление протоколов встреч._ | **Стратегические проекты**<br>_Пример: Создание агента для анализа рынка и конкурентов._ |
| **Низкое влияние на бизнес** | **Дополнительные улучшения (Fill-ins)**<br>_Пример: Скилл для заказа еды в офис._ | **Задачи, которые стоит отложить (Thankless Tasks)**<br>_Пример: Интеграция с устаревшей легаси-системой без API._ |

**Как работать с матрицей:**

1.  Составьте список потенциальных задач (скиллов) для вашего агента.
2.  Для каждой задачи оцените сложность ее технической реализации (нужны ли сложные интеграции, есть ли API и т.д.).
3.  Для каждой задачи оцените, какой эффект она окажет на вашу личную продуктивность или на бизнес-процессы компании.
4.  Разместите задачи в соответствующих квадрантах матрицы.
5.  Начинайте работу с квадранта «Быстрые победы».
